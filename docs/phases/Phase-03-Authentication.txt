Design Decisions:
  - Only @org.com emails allowed — validated before touching DB
  - bcrypt salt rounds = 12 (good balance of security vs. performance)
  - JWT stored in HTTP-only cookie named "authToken"
  - secure: true cookie flag only in production (allows local dev over HTTP)
  - Login error is generic "Invalid credentials." to prevent user enumeration
  - Validation collects ALL field errors before throwing (not fail-fast per field)
  - errorHandler now handles UniqueConstraintError (409) and ValidationError (400)
  - Repository has two separate finders: findByEmail (no hash) vs
    findByEmailWithPassword (with hash) to make accidental exposure impossible

Endpoints:
  POST /api/rest/onboardings/register
  POST /api/rest/onboardings/login
  POST /api/rest/onboardings/logout
