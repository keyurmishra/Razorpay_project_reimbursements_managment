Design Decisions:
  - passwordHash field name (not "password") to signal it stores a hash
  - underscored: true  →  passwordHash maps to password_hash column
  - defaultScope excludes passwordHash from all queries automatically
  - withPassword scope used only in auth service for login verification
  - Roles: EMP (default) | RM | APE | CFO  stored as PostgreSQL ENUM
  - Migration down() also drops the ENUM type (PostgreSQL-specific cleanup)
  - Indexes: unique on email, plain index on role
